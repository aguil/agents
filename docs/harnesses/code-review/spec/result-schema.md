# result.json Schema

Each harness run writes `result.json` (filtered, actionable output) and
`result.raw.json` (orchestration output before report filtering) under
`.agents-code-review/runs/<run-id>/`.

## Top-level fields

| Field               | Type                                            | Description                                  |
| ------------------- | ----------------------------------------------- | -------------------------------------------- |
| `runId`             | string                                          | Harness run ID (also in `metadata.run_id`)   |
| `status`            | `"passed" \| "warnings" \| "failed" \| "error"` | Overall result                               |
| `findings`          | Finding[]                                       | Deduped, actionable findings (verified only) |
| `artifacts`         | object                                          | Paths generated by the run                   |
| `metadata`          | object                                          | Adapter and triage metadata (see below)      |
| `reportPath`        | string                                          | Absolute path to `report.md`                 |
| `contextBundlePath` | string                                          | Absolute path to `context/bundle.json`       |

## `status` values

| Value      | Meaning                                                          |
| ---------- | ---------------------------------------------------------------- |
| `passed`   | No actionable findings                                           |
| `warnings` | One or more warning-severity findings                            |
| `failed`   | One or more error/critical findings                              |
| `error`    | Fatal harness error (adapter spawn failure, orchestration error) |

Exit code is `1` when status is `error`; `0` otherwise.

## Finding shape

Actionable findings additionally require substantive validation details (not
just a bare assertion). Finding dedupe uses a canonical fingerprint (role +
location + normalized semantic signature) to reduce rewording churn across runs.

| Field        | Type    | Description                                        |
| ------------ | ------- | -------------------------------------------------- |
| `role`       | string  | Reviewer role that produced the finding            |
| `severity`   | string  | Finding severity                                   |
| `title`      | string  | Short finding title                                |
| `file`       | string? | Source file path (present for anchorable findings) |
| `line`       | number? | Source line (present for anchorable findings)      |
| `evidence`   | string  | Supporting evidence                                |
| `validation` | object  | Validation status and details                      |

Only findings with `file` + `line` are posted as inline GitHub review comments
in pending-review mode.

## `metadata` fields

### Harness and run

| Field                 | Description                                             |
| --------------------- | ------------------------------------------------------- |
| `run_id`              | Harness run ID                                          |
| `triage`              | Selected triage tier (`trivial`, `lite`, `full`)        |
| `timed_out_roles`     | Comma-separated roles that timed out                    |
| `failed_roles`        | Comma-separated roles that failed                       |
| `completed_roles`     | Comma-separated roles that completed                    |
| `strict_mode`         | `"true"` when strict mode is enabled                    |
| `vcs_mode`            | Detected workspace mode (`jj`, `git`, or `unknown`)     |
| `context_source`      | `"live"` (collected) or `"replay"` (`--context-bundle`) |
| `context_fingerprint` | Short hash for context-equivalence checks               |
| `deterministic_mode`  | `"true"` when deterministic profile is enabled          |

### Config harness

Present on every config-declared code-review run.

| Field                              | Description                                                                 |
| ---------------------------------- | --------------------------------------------------------------------------- |
| `config_harness_source`            | Resolved source layer: `explicit`, `workspace`, `user-global`, or `package` |
| `config_harness_agents_dir`        | Absolute `.agents` directory that supplied the harness                      |
| `config_harness_package_version`   | Version of the running `@aguil/agents` package                              |
| `config_harness_installed_version` | Version marker from user-global or explicit installs, when present          |
| `config_harness_version_drift`     | `"true"` when installed version differs from package version                |

### Consensus

| Field                        | Description                             |
| ---------------------------- | --------------------------------------- |
| `consensus_runs`             | Number of passes requested              |
| `consensus_mode`             | `"off"` or `"intersection"`             |
| `consensus_dropped_findings` | Count of findings filtered by consensus |

### PR context

| Field                  | Description                                                      |
| ---------------------- | ---------------------------------------------------------------- |
| `pr_number`            | PR number from `--pr` or implicit `gh pr view` discovery         |
| `pr_reviewed_head_sha` | Remote PR head OID captured during context collection            |
| `pr_reviewed_at`       | ISO-8601 timestamp when PR patch context was collected           |
| `pr_posting_head_sha`  | PR head SHA seen immediately before posting (`--pending-review`) |
| `pr_head_diverged`     | `"true"` if posting happened after PR head changed               |

### Adapter-specific

| Field prefix | Adapter                                                            |
| ------------ | ------------------------------------------------------------------ |
| `opencode_*` | OpenCode model/runtime settings and detected executable version    |
| `claude_*`   | Claude Code model/runtime settings and detected executable version |
